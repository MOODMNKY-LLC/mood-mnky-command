## Getting Started with Development

To begin contributing to or building with the MOOD MNKY ecosystem, follow these steps:

### 1. Set Up Your Environment

<CodeGroup>
  ```bash Clone the repository
  git clone https://github.com/mood-mnky/MNKY-REPO.git
  cd MNKY-REPO
  ```

  ```bash Install dependencies
  pnpm install
  ```

  ```bash Run initial build
  pnpm build
  ```

  ```bash Start development server
  pnpm dev
  ```
</CodeGroup>

### 2. Monorepo Structure

Our codebase follows a clean, organized monorepo structure using Turborepo:

```
MNKY-REPO/
├── apps/                 # User-facing applications
│   ├── web/              # Main web application (Next.js)
│   ├── dashboard/        # User dashboard (Next.js)
│   ├── docs/             # Documentation site (Mintlify)
│   └── admin/            # Admin portal (Next.js)
├── packages/             # Shared libraries and utilities
│   ├── ui/               # Shared UI components
│   ├── agent-core/       # Shared agent infrastructure
│   ├── api-client/       # API client libraries
│   └── utils/            # Shared utilities
├── agents/               # Agent definitions and configurations
├── content/              # Content libraries and assets
├── docs/                 # Documentation (Mintlify)
└── infra/                # Infrastructure and deployment
```

## Core Development Areas

The MOOD MNKY development ecosystem is organized around these key areas:

### Frontend Development

Our frontend is built with Next.js, TypeScript, Tailwind CSS, and ShadCN UI components. We follow modern best practices including:

- Server Components for improved performance
- Type safety throughout the application
- Component-driven development with reusable UI elements
- Mobile-first responsive design
- Accessibility as a core consideration

## Development Workflows

### Code Organization

We follow these principles for code organization:

<Steps>
  <Step title="Feature-based organization">
    Group related files by feature rather than by file type
  </Step>
  <Step title="Shared abstractions">
    Extract common patterns into shared packages
  </Step>
  <Step title="Clear boundaries">
    Maintain clear interfaces between system components
  </Step>
  <Step title="Progressive enhancement">
    Build core functionality first, then enhance with advanced features
  </Step>
</Steps>

## Common Development Tasks

Here are some common development tasks you might perform with the MOOD MNKY ecosystem:

### Working with Apps

<CodeGroup>
  ```bash Start a specific app
  pnpm dev --filter=web
  ```

  ```bash Build a specific app
  pnpm build --filter=dashboard
  ```

  ```bash Run tests for an app
  pnpm test --filter=web
  ```
</CodeGroup>

### Working with Packages

<CodeGroup>
  ```bash Develop a shared package
  pnpm dev --filter=@repo/ui
  ```

  ```bash Build all packages
  pnpm build --filter='./packages/*'
  ```

  ```bash Update dependencies across workspace
  pnpm update -r
  ```
</CodeGroup>

### Working with Agents

<CodeGroup>
  ```bash Build agent flows
  pnpm agent:build
  ```

  ```bash Test agent capabilities
  pnpm agent:test
  ```

  ```bash Deploy agent updates
  pnpm agent:deploy
  ```
</CodeGroup>
