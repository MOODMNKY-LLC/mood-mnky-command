# Get Group By Name

This endpoint returns detailed information about a Bungie.net group (clan or group) based on its name and type. This is particularly useful for finding clans by their name without knowing their ID.

## Authentication

This endpoint requires:
- A valid Bungie.net API key
- No OAuth authentication is required for public groups

## Path Parameters

<ParamField path="groupName" type="string" required>
  The name of the group to retrieve
</ParamField>

<ParamField path="groupType" type="integer" required>
  The type of group to retrieve:
  - 0: General
  - 1: Clan
</ParamField>

<RequestExample>
```bash
curl -X GET "https://www.bungie.net/Platform/GroupV2/Name/MNKY%20Clan/1/" \
  -H "X-API-Key: your-api-key-here"
```

```javascript
const fetchGroupByName = async (groupName, groupType) => {
  const encodedName = encodeURIComponent(groupName);
  
  const response = await fetch(
    `https://www.bungie.net/Platform/GroupV2/Name/${encodedName}/${groupType}/`,
    {
      headers: {
        'X-API-Key': 'your-api-key-here'
      }
    }
  );
  
  return await response.json();
};
```
</RequestExample>

## Usage in MOOD MNKY

The Get Group By Name endpoint is used in MOOD MNKY applications for:

1. **Clan Search** - Allowing users to find and join clans by name rather than requiring an ID
2. **Clan Profiles** - Displaying detailed information about a clan, including its banner, membership options, and member count
3. **Clan Management Tools** - Retrieving clan details for administrative purposes
4. **Social Integration** - Finding allied clans and potential community partnerships

### Integration Example

```javascript
// Clan search component
import React, { useState } from 'react';

function ClanSearch() {
  const [clanName, setClanName] = useState('');
  const [searchResults, setSearchResults] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const searchClan = async () => {
    if (!clanName.trim()) {
      setError('Please enter a clan name');
      return;
    }

    setLoading(true);
    setError(null);

    try {
      const encodedName = encodeURIComponent(clanName.trim());
      const response = await fetch(
        `https://www.bungie.net/Platform/GroupV2/Name/${encodedName}/1/`,
        {
          headers: {
            'X-API-Key': process.env.BUNGIE_API_KEY
          }
        }
      );

      const data = await response.json();

      if (data.ErrorCode !== 1) {
        throw new Error(data.Message || 'Failed to find clan');
      }

      setSearchResults(data.Response);
    } catch (err) {
      setError(err.message || 'An error occurred while searching for the clan');
    } finally {
      setLoading(false);
    }
  };

  const renderClanDetails = () => {
    if (!searchResults) return null;

    const { detail, founder } = searchResults;
    
    // Get clan banner URL from the clan banner data
    const clanBannerUrl = generateClanBannerUrl(detail.clanInfo.clanBannerData);

    return (
      <div className="clan-details">
        <div className="clan-header">
          <img src={clanBannerUrl} alt="Clan Banner" className="clan-banner" />
          <div className="clan-info">
            <h2>{detail.name} [{detail.clanInfo.clanCallsign}]</h2>
            <p className="clan-motto">{detail.motto}</p>
            <p className="clan-founded">
              Founded {new Date(detail.creationDate).toLocaleDateString()}
            </p>
            <p className="clan-members">{detail.memberCount} members</p>
          </div>
        </div>
        
        <div className="clan-about">
          <h3>About</h3>
          <p>{detail.about || 'No description provided.'}</p>
        </div>
        
        <div className="clan-founder">
          <h3>Founder</h3>
          <p>{founder.bungieNetUserInfo.bungieGlobalDisplayName}#{founder.bungieNetUserInfo.bungieGlobalDisplayNameCode}</p>
        </div>
        
        <div className="clan-join">
          {detail.membershipOption === 1 ? (
            <button className="join-button">Request to Join</button>
          ) : detail.membershipOption === 0 ? (
            <button className="join-button">Apply to Join (Requires Approval)</button>
          ) : (
            <span className="closed-clan">This clan is not accepting new members</span>
          )}
        </div>
      </div>
    );
  };

  return (
    <div className="clan-search-container">
      <h1>Find a Clan</h1>
      
      <div className="search-bar">
        <input
          type="text"
          value={clanName}
          onChange={(e) => setClanName(e.target.value)}
          placeholder="Enter clan name..."
          className="clan-search-input"
        />
        <button 
          onClick={searchClan} 
          disabled={loading}
          className="search-button"
        >
          {loading ? 'Searching...' : 'Search'}
        </button>
      </div>
      
      {error && <div className="error-message">{error}</div>}
      
      {loading ? (
        <div className="loading">Searching for clan...</div>
      ) : (
        renderClanDetails()
      )}
    </div>
  );
}

// Helper function to generate clan banner URL from banner data
function generateClanBannerUrl(bannerData) {
  if (!bannerData) return '/default-banner.png';
  
  return `https://www.bungie.net/img/UserThemes/ClanBanners/Banner_${bannerData.gonfalonId}_${bannerData.gonfalonColorId}_${bannerData.gonfalonDetailId}_${bannerData.gonfalonDetailColorId}_${bannerData.decalId}_${bannerData.decalColorId}_${bannerData.decalBackgroundColorId}.png`;
}

export default ClanSearch;
```

## Common Issues

- **Exact Name Required**: The search is case-sensitive and requires the exact clan name. Partial matches will not work.
- **URL Encoding**: Ensure the clan name is properly URL encoded, especially for names with spaces or special characters.
- **Similar Names**: Multiple clans may have similar names. The endpoint only returns one result for the exact match.
- **Clan vs Group**: Be sure to use the correct `groupType` parameter (1 for clans, 0 for groups).
- **Rate Limiting**: This endpoint is subject to Bungie's rate limiting. Monitor the `ThrottleSeconds` value in responses.
- **Private Groups**: Some group information may be limited for private groups.
