# Get Destiny 2 Profile

This endpoint returns Destiny 2 profile information for the specified account. It allows you to request specific components of profile data, giving you precise control over what information is returned.

## Authentication

This endpoint requires:
- A valid Bungie.net API key
- For private profiles, OAuth 2.0 authentication with the `ReadDestinyInventoryAndVault` scope

## Path Parameters

<ParamField path="membershipType" type="integer" required>
  A valid Destiny 2 platform identifier:
  - 1: Xbox
  - 2: PlayStation
  - 3: Steam
  - 4: Blizzard (legacy)
  - 5: Stadia (legacy)
  - 6: Epic Games
  - 10: Demon
  - 254: BungieNext
</ParamField>

<ParamField path="destinyMembershipId" type="string" required>
  Destiny membership ID of the account whose profile you want to access
</ParamField>

## Query Parameters

<ParamField query="components" type="array" required>
  A comma-separated list of components to return (as a string parameter). See [Components](#components) for the list of available components.
</ParamField>

```bash
curl -X GET "https://www.bungie.net/Platform/Destiny2/3/Profile/4611686018429783292/?components=100,200,201,205" \
  -H "X-API-Key: your-api-key-here" \
  -H "Authorization: Bearer your-oauth-token"
```

```javascript
const fetchProfile = async (membershipType, destinyMembershipId, components, accessToken) => {
  const componentsParam = components.join(',');
  
  const response = await fetch(
    `https://www.bungie.net/Platform/Destiny2/${membershipType}/Profile/${destinyMembershipId}/?components=${componentsParam}`,
    {
      headers: {
        'X-API-Key': 'your-api-key-here',
        'Authorization': accessToken ? `Bearer ${accessToken}` : undefined
      }
    }
  );
  
  return await response.json();
};
```

## Usage in MOOD MNKY

The Get Profile endpoint is a cornerstone of the MOOD MNKY ecosystem, providing critical character and inventory data for:

1. **Inventory Management** - Fetching equipped items, vault contents, and character inventories
2. **Character Selection** - Displaying available characters with their class, power level, and emblems
3. **Item Stat Tracking** - Retrieving detailed item stats, socket configurations, and masterwork status
4. **Collections & Triumphs** - Tracking player collections and triumph progress
5. **Season Progress** - Monitoring season pass level and event participation

### Integration Example

```javascript
// Retrieve and display character information
async function loadCharacters(membershipType, membershipId, accessToken) {
  try {
    // Request only the components we need
    const components = ['100', '200']; // Profile and Characters components
    
    const response = await fetch(
      `https://www.bungie.net/Platform/Destiny2/${membershipType}/Profile/${membershipId}/?components=${components.join(',')}`,
      {
        headers: {
          'X-API-Key': process.env.BUNGIE_API_KEY,
          'Authorization': accessToken ? `Bearer ${accessToken}` : undefined
        }
      }
    );
    
    const data = await response.json();
    
    if (data.ErrorCode !== 1) {
      throw new Error(`Failed to get profile data: ${data.Message}`);
    }
    
    const { profile, characters } = data.Response;
    
    if (!profile?.data || !characters?.data) {
      throw new Error('Missing required profile data components');
    }
    
    // Get class definitions to map class hashes to readable names
    const classDefinitions = await getDestinyDefinitions('DestinyClassDefinition');
    
    // Format character data for display
    const characterData = Object.values(characters.data).map(character => {
      const classDefinition = classDefinitions[character.classHash];
      
      return {
        characterId: character.characterId,
        lastPlayed: new Date(character.dateLastPlayed),
        light: character.light,
        className: classDefinition?.displayProperties.name || 'Unknown Class',
        emblemBackgroundPath: `https://www.bungie.net${character.emblemBackgroundPath}`,
        minutesPlayed: character.minutesPlayedTotal,
        level: character.baseCharacterLevel
      };
    });
    
    // Sort characters by last played date (most recent first)
    return characterData.sort((a, b) => b.lastPlayed - a.lastPlayed);
  } catch (error) {
    console.error('Error loading characters:', error);
    throw error;
  }
}

// Example for loading inventory
async function loadInventory(membershipType, membershipId, characterId, accessToken) {
  try {
    // Request character equipment and instances for detailed item information
    const components = ['205', '300', '304', '305']; // Equipment, instances, stats, sockets
    
    const response = await fetch(
      `https://www.bungie.net/Platform/Destiny2/${membershipType}/Profile/${membershipId}/?components=${components.join(',')}`,
      {
        headers: {
          'X-API-Key': process.env.BUNGIE_API_KEY,
          'Authorization': `Bearer ${accessToken}`
        }
      }
    );
    
    const data = await response.json();
    
    if (data.ErrorCode !== 1) {
      throw new Error(`Failed to get inventory data: ${data.Message}`);
    }
    
    const { characterEquipment, itemComponents } = data.Response;
    
    if (!characterEquipment?.data || !characterEquipment.data[characterId]) {
      throw new Error(`No equipment data found for character ${characterId}`);
    }
    
    // Get item definitions for display
    const itemDefinitions = await getDestinyDefinitions('DestinyInventoryItemDefinition');
    
    // Format equipment data for display
    const equipment = characterEquipment.data[characterId].items.map(item => {
      const itemDefinition = itemDefinitions[item.itemHash];
      const instanceData = itemComponents.instances.data[item.itemInstanceId];
      
      return {
        itemHash: item.itemHash,
        instanceId: item.itemInstanceId,
        name: itemDefinition?.displayProperties.name || 'Unknown Item',
        icon: itemDefinition?.displayProperties.icon ? `https://www.bungie.net${itemDefinition.displayProperties.icon}` : null,
        itemTypeDisplayName: itemDefinition?.itemTypeDisplayName || '',
        powerLevel: instanceData?.primaryStat?.value || 0,
        damageType: instanceData?.damageType || 0,
        energy: instanceData?.energy || null,
        isExotic: itemDefinition?.inventory.tierType === 6
      };
    });
    
    // Group equipment by bucket (slot)
    const slots = {};
    equipment.forEach(item => {
      const bucketHash = item.bucketHash;
      if (!slots[bucketHash]) {
        slots[bucketHash] = [];
      }
      slots[bucketHash].push(item);
    });
    
    return { equipment, slots };
  } catch (error) {
    console.error('Error loading inventory:', error);
    throw error;
  }
}
```

## Common Issues

- **Component Selection**: Requesting too many components at once can lead to longer response times or timeouts. Only request components you need.
- **Privacy Settings**: If a profile has private settings, certain components may not be accessible without proper authentication.
- **Rate Limiting**: The Bungie API has rate limits. Monitor the `ThrottleSeconds` value in responses.
- **Instanced Data**: For detailed item information (stats, perks, etc.), you must include the appropriate ItemInstance components.
- **Manifest Dependency**: Many returned values are hashes that must be resolved using the Destiny 2 Manifest.
- **Response Size**: For active players with many items, responses can be large. Implement proper pagination or filtering on your end.
- **Character ID Requirement**: Some components require a valid character ID, which you must obtain from the Characters component first.
