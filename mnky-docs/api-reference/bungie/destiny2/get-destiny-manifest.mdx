# Get Destiny 2 Manifest

The Destiny 2 Manifest contains reference data for all static items, activities, locations, and other game elements. This endpoint returns the current version of the manifest and URLs to download the various components.

## Authentication

This endpoint requires an API key but does not require OAuth authentication.

<RequestExample>
```bash
curl -X GET "https://www.bungie.net/Platform/Destiny2/Manifest/" \
  -H "X-API-Key: your-api-key-here"
```
</RequestExample>

## Usage in MOOD MNKY

Within the MOOD MNKY ecosystem, this endpoint is used to:

1. Keep game data up-to-date across all applications
2. Download localized content for multi-language support
3. Reference game items, activities, and other entities by hash ID
4. Access game images and other assets through the content paths

### Integration Example

```javascript
// Example of fetching the manifest in a MOOD MNKY application
async function getDestinyManifest() {
  const response = await fetch('https://www.bungie.net/Platform/Destiny2/Manifest/', {
    headers: {
      'X-API-Key': process.env.BUNGIE_API_KEY
    }
  });
  
  const data = await response.json();
  
  if (data.ErrorCode === 1) {
    return data.Response;
  } else {
    throw new Error(`Failed to get manifest: ${data.Message}`);
  }
}
```

## Common Issues

- **Rate Limiting**: Bungie enforces rate limits on API calls. If you encounter `ThrottleSeconds > 0`, wait the specified time before making another request.
- **API Key**: Ensure your API key is valid and has not expired.
- **Manifest Size**: The manifest database files can be large. Implement proper caching to avoid downloading them frequently.
