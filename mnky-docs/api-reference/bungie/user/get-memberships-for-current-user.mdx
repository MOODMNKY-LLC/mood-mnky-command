# Get Memberships for Current User

This endpoint returns a list of all membership profiles linked to the currently authenticated OAuth user. The response includes accounts across all platforms where the user has played Destiny 2.

## Authentication

This endpoint requires:
- A valid Bungie.net API key
- OAuth 2.0 authentication with the `ReadBasicUserProfile` scope

<RequestExample>
```bash
curl -X GET "https://www.bungie.net/Platform/User/GetMembershipsForCurrentUser/" \
  -H "X-API-Key: your-api-key-here" \
  -H "Authorization: Bearer your-oauth-token"
```

```javascript
const fetchMemberships = async (accessToken) => {
  const response = await fetch(
    'https://www.bungie.net/Platform/User/GetMembershipsForCurrentUser/',
    {
      headers: {
        'X-API-Key': 'your-api-key-here',
        'Authorization': `Bearer ${accessToken}`
      }
    }
  );
  
  return await response.json();
};
```
</RequestExample>

## Usage in MOOD MNKY

This endpoint is vital in the MOOD MNKY ecosystem for:

1. **User Authentication Flow** - When a user signs in with OAuth, this endpoint helps identify all linked platforms
2. **Cross-platform Profile Management** - Allows users to manage inventories across different gaming platforms
3. **Platform Selection** - Enables MOOD MNKY to provide platform selection when interacting with Destiny items
4. **Cross-save Detection** - Helps detect and manage cross-save status for the user
5. **Display Name Resolution** - Gets the proper display names to show across the interface

### Integration Example

```javascript
// Example of handling multi-platform profiles in MOOD MNKY
async function initializeUserProfiles(accessToken) {
  try {
    // Fetch all memberships for the authenticated user
    const membershipsResponse = await fetch(
      'https://www.bungie.net/Platform/User/GetMembershipsForCurrentUser/',
      {
        headers: {
          'X-API-Key': process.env.BUNGIE_API_KEY,
          'Authorization': `Bearer ${accessToken}`
        }
      }
    );
    
    const data = await membershipsResponse.json();
    
    if (data.ErrorCode !== 1) {
      throw new Error(`Failed to get memberships: ${data.Message}`);
    }
    
    // Extract the memberships from the response
    const { destinyMemberships, primaryMembershipId, bungieNetUser } = data.Response;
    
    // Handle cross-save situation
    let activeMembership = null;
    
    if (primaryMembershipId) {
      // If cross-save is enabled, find the primary membership
      activeMembership = destinyMemberships.find(
        membership => membership.membershipId === primaryMembershipId
      );
    }
    
    // If no active membership is found (or cross-save is not enabled),
    // let the user choose from available platforms
    if (!activeMembership && destinyMemberships.length > 0) {
      // For this example, just use the first membership
      activeMembership = destinyMemberships[0];
    }
    
    // Store user profile information for later use
    return {
      bungieNetUser,
      destinyMemberships,
      primaryMembership: activeMembership,
      hasCrossSave: !!primaryMembershipId,
      selectedMembershipType: activeMembership ? activeMembership.membershipType : null,
      selectedMembershipId: activeMembership ? activeMembership.membershipId : null,
      displayName: bungieNetUser.cachedBungieGlobalDisplayName,
      displayNameCode: bungieNetUser.cachedBungieGlobalDisplayNameCode
    };
  } catch (error) {
    console.error('Error initializing user profiles:', error);
    throw error;
  }
}

// Example component usage
function PlatformSelector({ userProfiles, onSelectPlatform }) {
  // Map platform types to readable names and icons
  const platformInfo = {

      </div>
    </div>
  );
}
```

## Common Issues

- **Token Authentication**: This endpoint requires a valid OAuth token with appropriate scopes.
- **Cross-save Complexity**: For users with cross-save enabled, ensure you use the `primaryMembershipId` to identify their active profile.
- **Multiple Accounts**: A user may have multiple accounts on the same platform in rare cases. The `bungieGlobalDisplayName` and `bungieGlobalDisplayNameCode` fields help identify the unique user.
- **Platform Migration**: Users who have migrated between platforms may have multiple membership entries.
- **Privacy Settings**: Some membership information may be limited by user privacy settings.
