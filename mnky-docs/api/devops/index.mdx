## Authentication

All DevOps API endpoints require authentication. We support two authentication methods:

1. **API Key Authentication**
   - Required for all production endpoints
   - Keys are managed through our secure key management system
   - Include the API key in the `Authorization` header:
     ```bash
     Authorization: Bearer your-api-key
     ```

2. **JWT Token Authentication**
   - Used for internal service-to-service communication
   - Tokens are automatically managed by our service mesh
   - Short-lived tokens with automatic rotation

## Rate Limiting

To ensure service stability and fair usage:

- **Ollama Service**
  - 100 requests per minute per API key for generation endpoints
  - 1000 requests per minute per API key for other endpoints
  - Burst limits may apply

- **Flowise Service**
  - 60 requests per minute per API key for chatflow execution
  - 300 requests per minute per API key for management endpoints
  - Additional limits based on subscription tier

## Error Handling

All API endpoints follow consistent error response formats:

```json
{
  "error": {
    "code": "error_code",
    "message": "Human-readable error message",
    "details": {
      "additional": "error specific details"
    }
  }
}
```

Common HTTP status codes:
- `400` - Bad Request
- `401` - Unauthorized
- `403` - Forbidden
- `404` - Not Found
- `429` - Too Many Requests
- `500` - Internal Server Error
